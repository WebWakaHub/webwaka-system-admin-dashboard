## Document Purpose

This document provides a **decision-by-decision authority table** that explicitly defines what the **Founder**, **Founder Agent**, **Chief of Staff**, and **Execution Agents** may do.

**Authority Markings:**
- ✅ **YES** - Authorized to perform this action independently
- ❌ **NO** - Not authorized to perform this action
- ⚠️ **CONDITIONAL** - Authorized only under specific conditions (conditions specified)

**This matrix is binding and must be enforced by all agents and systems.**

---

## Authority Hierarchy

```
Human Founder (Ultimate Authority)
  ↓ (explicit delegation)
Founder Agent (Delegated Authority)
  ↓ (coordination & enforcement)
Chief of Staff (Operational Coordination)
  ↓ (execution)
Execution Agents (webwakaagent1-10)
```

## Delegation Scope Summary

### Founder Agent Current Delegations (Phase 1)

**✅ DELEGATED:**
1. Draft Founder Decisions (for Founder approval)
2. Approve Phase 1 document production outputs
3. Verify agent compliance (Phase 1)
4. Grant agent authorization (Phase 1 activities)
5. Coordinate workflows across all departments
6. Resolve conflicts between agents
7. Enforce governance compliance
8. Issue compliance warnings
9. Halt execution on governance violations
10. Conduct weekly compliance reviews
11. Approve Phase 1 completion (when criteria met)
12. Define Phase 1 operational processes
13. Clarify governance ambiguities (operational scope)

**❌ NOT DELEGATED:**
1. Issue Founder Decisions
2. Modify existing Founder Decisions
3. Revoke Founder Decisions
4. Change governance authority structure
5. Delegate authority to others
6. Authorize spending or financial commitments
7. Enter legal agreements or contracts
8. Terminate agents or systems permanently
9. Change Phase scope definitions
10. Override governance rules (except emergency)
11. Define platform vision, mission, or institutional principles
12. Approve product strategy without explicit delegation
13. Approve architecture decisions without explicit delegation
14. Approve technology stack without explicit delegation

## How to Use This Matrix

### For Agents
1. Locate the action you want to perform in the matrix
2. Check your authority level (Founder Agent, Chief of Staff, or Executor)
3. Verify if you have YES, NO, or CONDITIONAL authority
4. If CONDITIONAL, check the conditions carefully
5. If NO or conditions not met, escalate to higher authority
6. If ambiguous, HALT and request clarification from Founder

### For Founder
1. Reference this matrix when delegating authority
2. Update this matrix when changing delegations (via Founder Decision)
3. Enforce this matrix when reviewing agent actions
4. Revoke delegations by updating this matrix

### For Chief of Staff
1. Enforce this matrix across all agents
2. Verify agent actions comply with this matrix
3. Escalate violations to Founder or Founder Agent
4. Request matrix updates when ambiguities arise
