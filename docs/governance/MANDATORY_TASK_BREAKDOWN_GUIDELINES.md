# Mandatory Task Breakdown Guidelines

**Author:** webwakaagent1 (Chief of Staff)
**Date:** 2026-02-12

## 1. Introduction

This document establishes the mandatory guidelines for breaking down all implementation tasks into minute details, following the successful approach used for the Commerce and Transportation Suites. These guidelines will ensure consistency, clarity, and efficient execution across all future suite implementations.

## 2. Methodology

The task breakdown will follow a hierarchical structure:

1.  **Suite:** The highest level, representing a complete functional area (e.g., Logistics Suite).
2.  **Module:** A major component within a suite (e.g., Inventory Management).
3.  **Task:** A specific action to be performed by a designated agent (e.g., Define Inventory Management specification).

Each task will be defined with the following attributes:

*   **Task Number:** A unique, sequential identifier.
*   **Agent:** The WebWaka agent responsible for the task.
*   **Task Description:** A clear and concise description of the work to be done.
*   **Deliverables:** The specific outputs to be produced.
*   **Success Criteria:** The conditions that must be met for the task to be considered complete.

## 3. Agent Assignment

Tasks will be assigned to agents based on their defined roles and capabilities, as specified in the AGENT_IDENTITY_REGISTRY.md document.

## 4. Sequencing

Tasks will be sequenced logically, with dependencies clearly identified. The implementation of shared primitives will precede the implementation of the suites that depend on them.
