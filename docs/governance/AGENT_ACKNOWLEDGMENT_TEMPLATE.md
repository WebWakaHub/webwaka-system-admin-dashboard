## TEMPLATE INSTRUCTIONS

### Placeholders to Replace:

1. **[AGENT_ID]**: Your agent ID (e.g., webwakaagent2, webwakaagent3, etc.)
2. **[ROLE_NAME]**: Your role name (e.g., Product & Platform Strategy Lead, Architecture & System Design Lead)
3. **[DEPARTMENT_NAME]**: Your department name (e.g., Product & Platform Strategy, Architecture & System Design)
4. **[YYYY-MM-DD]**: Current date in YYYY-MM-DD format (e.g., 2026-02-05)
5. **[Authority items]**: Copy from your agent context page (docs/agents/context/[AGENT_ID].md) under "Authority Scope" section
6. **[Specific prohibitions]**: Copy from your agent context page under "Explicit Prohibitions" section

### Format Requirements:

1. **Plain text only** - No bold, no italic, no markdown formatting in acknowledgement text
2. **Exact section titles** - Use "Identity & Role Acknowledgement", "Activation Confirmation", "Governance & Context Assimilation"
3. **Exact activation statement** - Use the 2-sentence format exactly as shown
4. **All 14 documents** - List all 14 mandatory documents by exact filename (do not add or remove any)
5. **Numbered list** - Use numbered list (1-14) for document list

### Common Mistakes to Avoid:

❌ Using **bold** or *italic* in acknowledgement text  
❌ Adding extra documents to the list  
❌ Removing documents from the list  
❌ Changing document filenames  
❌ Using different section titles  
❌ Adding extra text to activation confirmation  
❌ Using checkboxes (✅) in document list

### Verification Checklist:

Before committing your acknowledgment file, verify:

- [ ] All placeholders replaced with correct information
- [ ] No bold/italic formatting in acknowledgement text
- [ ] All 14 documents listed by exact filename
- [ ] Activation confirmation uses exact 2-sentence format
- [ ] File saved as `agent-acknowledgments/WEBWAKAAGENT[N]_ACKNOWLEDGMENT.md`
- [ ] Authority items copied from agent context page
- [ ] Prohibitions copied from agent context page
