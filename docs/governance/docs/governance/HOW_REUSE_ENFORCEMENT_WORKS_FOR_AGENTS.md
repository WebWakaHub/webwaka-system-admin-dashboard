# How Reuse Enforcement Works for Agents

**Date:** 2026-02-14
**From:** Chief of Staff (webwakaagent1)
**To:** All Agents

---

This document explains the new Reuse Enforcement program and how it will affect your daily work. This is not a bureaucracy; it is a system designed to make your work easier, safer, and more impactful.

## Why We Are Doing This

We are building a single, unified platform, not a collection of disconnected products. Reusing existing capabilities is essential to achieving this vision. It reduces rework, increases velocity, and allows us to build a more stable and scalable system.

## What is Changing

Before you start any new task that involves building a feature, creating a module, or adding a capability, you must now complete a **Reuse Declaration**.

## The Reuse Declaration Process

1.  **Search the Registry:** Before you do anything else, search the `REUSE_REGISTRY_V0.md` to see if a capability that meets your needs already exists.
2.  **Complete the Declaration:** Fill out the `REUSE_DECLARATION.md` template. This is a simple form that asks you to document your search and your decision.
3.  **Submit for Approval:**
    -   If you are reusing or extending an existing capability, submit the declaration to the Chief of Staff.
    -   If you need to build a new capability, submit the declaration to the Architecture Review Board (ARB).

**That's it.** This is not meant to be a heavyweight process. It is a simple, mechanical check to ensure that we are not reinventing the wheel.

## How This Helps You

-   **You are safer.** This system protects you from building the wrong thing and facing rework.
-   **You will move faster.** By reusing existing capabilities, you can focus on building new, innovative features, not re-implementing the basics.
-   **Help exists.** The ARB and Chief of Staff are here to help you navigate the system, not to punish you.
-   **Escalation is structured.** If you disagree with a decision, there is a clear and rapid process for appealing it.

This is a positive change for the platform and for all of us. It will allow us to build better software, faster. If you have any questions, please do not hesitate to reach out to me.
